配置网卡IP地址 

	NetworkManager服务 
	
		配置工具：nmcli 
得

网卡配置文件：

	/etc/sysconfig/network-scripts/ifcfg-XXX
	
		eth0   ---- ifcfg-eth0 
		ens22  ---- ifcfg-ens22

		
方法一) 编辑网卡配置文件

	# vim /etc/sysconfig/network-scripts/ifcfg-ens33 

		DEVICE=ens33
		NAME=ens33
		TYPE=Ethernet
		ONBOOT=yes
		BOOTPROTO=none
		IPADDR=10.1.1.1
		PREFIX=24
		GATEWAY=10.1.1.254
		DNS1=114.114.114.114
		DNS2=202.106.0.20

	# nmcli connection reload
	# nmcli connection up ens33 
	# ifconfig 


测试网络通信

	# ping IP地址 
	
	
查看

1、查看IP地址

	# ifconfig 
	# ip addr show 
	
2、查看DNS服务器地址 

	[root@localhost ~]# cat /etc/resolv.conf 
	# Generated by NetworkManager
	nameserver 202.106.0.20
	nameserver 114.114.114.114

3、查看网关/路由表 

	[root@localhost ~]# route -n 


	

方法二) nmcli工具配置网卡 

1、删除自动生成的网卡配置文件 

	# nmcli connection show 
	# nmcli connection delete <UUID>
	# nmcli connection show 


2、创建网卡配置文件 

	# nmcli connection add type ethernet ifname ens37 con-name ens37 
	# nmcli connection show 
	
	
3、配置网络参数 

	# nmcli connection modify ens37 ipv4.addresses "172.16.1.1/24"
	# nmcli connection modify ens37 ipv4.gateway "172.16.1.254"
	# nmcli connection modify ens37 ipv4.dns "8.8.8.8"
	# nmcli connection modify ens37 +ipv4.dns "4.4.4.4"
	
	# nmcli connection modify ens37 ipv4.method manual 
	

	# nmcli connection reload
	# nmcli connection up ens37

国际电气联盟组织  IEEE 

	OSI七层模型 
	
		应用层、表示层、会话层、传输层、网络层、数据链路层、物理层 
		
	作用：
	
		应用层：提供用于网络通信的应用程序; 提供人与网络间的接口
		
		表示层：数据格式的转换、压缩/解压缩、加密、解密

		会话session层：建立会话、维护会话、断开会话；登录、退出

		传输层：提供应用程序到应用程序的数据通信 

		网络层：提供不同网络间的通信 		
		
		数据链路层：提供点到点的通信		
		
		物理层：提供物理链路的连接


	TCP/IP 五层模型：
	
		应用层、传输层、网络层、数据链路层、物理层 


	各层代表性的协议：
	
		应用层： 
		
			通过浏览器访问网站：
				http	超文本传输协议 
				https	

			
			ftp	   文件传输协议  	实现文件传输
	
			smtp/pop3/imap4		邮件
			
				smtp：发送邮件
				pop3：接收邮件
				imap4：支持附件 


		传输层： 
		
			TCP 传输控制协议	
				可靠协议 
				
			UDP 用户数据报协议 
				不可靠协议 
		

		网络层:
		
			IP：根据IP地址实现数据传输 
			
			ICMP: 实现网络通信的测试  ping 
			
			IGMP: 组播
			
			ARP：地址解析协议，根据已知IP地址，对应的MAC地址

TCP/UDP协议区别：

			
	1、TCP
	
		面向连接的协议 
		可靠的协议 (计时器)
			重传计时器  
			滑动窗口机制
	
		数据传输前：
			建立TCP连接，三次握手
				SYN   请求建立连接
				SYN + ACK    确认建立连接，对方主机发送的建立连接的请求
				ACK 	确认建立连接
				
			断开TCP连接，四次断开 

	2、UDP
	
		无连接的协议 
		不可靠的协议
